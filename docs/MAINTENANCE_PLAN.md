# Maintenance Plan

这个计划用于说明项目会如何长期维护，适合放在仓库中增强可信度。

## 维护目标

- 让算法示例更容易读懂
- 让每个示例都能运行和验证
- 让新手更容易参与贡献
- 让项目逐步支持中英文读者

## 每周维护节奏

- 检查新 Issue 和 PR
- 修正文档中的不清楚内容
- 给 1 到 2 个示例补充测试或说明
- 标记适合新手参与的问题

## 每月维护节奏

- 新增 1 到 3 个算法示例
- 更新算法索引
- 整理一篇学习文章或发布记录
- 复盘项目反馈并调整 Roadmap

## 质量标准

新增或修改示例时，尽量满足：

- 有题目说明
- 有核心思路
- 有可运行示例
- 有边界情况说明
- 重要算法有复杂度说明

## 适合新手参与的任务

- 补充输入输出样例
- 增加单元测试
- 改进注释
- 翻译文档
- 整理算法索引
- 修复错别字和格式问题

## 不接受的内容

- 刷 star、fork 或虚假活跃
- 没有说明来源的大段复制内容
- 与项目学习目标无关的功能
- 无法运行且没有说明的示例代码
