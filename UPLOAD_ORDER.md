# Upload Order

如果要手动把这些文件放到 GitHub，建议按下面顺序提交。

## 第一批：项目门面

- `README.md`
- `README_EN.md`
- `LICENSE`
- `CONTRIBUTING.md`
- `ROADMAP.md`

## 第二批：项目可信度

- `docs/ALGORITHM_INDEX.md`
- `docs/MAINTENANCE_PLAN.md`
- `docs/GROWTH_CHECKLIST.md`
- `docs/OPENAI_OSS_APPLICATION.md`

## 第三批：协作入口

- `.github/ISSUE_TEMPLATE/bug_report.md`
- `.github/ISSUE_TEMPLATE/algorithm_request.md`
- `.github/pull_request_template.md`

## 第四批：增长材料

- `PROMOTION.md`
- `GITHUB_ISSUES_TO_CREATE.md`

## 建议提交说明

```text
Improve open source project documentation
```

## 提交后检查

- 仓库首页是否能看到 README
- 许可证是否显示 MIT
- `docs` 目录是否能打开
- Issue 页面是否出现模板
- 新建 PR 时是否出现 PR 模板
